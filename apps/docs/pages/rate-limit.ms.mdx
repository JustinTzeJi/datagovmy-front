## Kadar Penghadan

Untuk memastikan penggunaan yang adil dan prestasi yang konsisten, kami menguatkuasakan had kadar pada bilangan permintaan yang boleh dibuat dalam tempoh masa tertentu. Ini juga memastikan sebanyak mungkin pengguna API mempunyai peluang untuk menggunakan API tanpa sebarang kelambatan.

Jika had tersebut diatasi, permintaan berikutnya akan gagal, dan anda akan menerima ralat `429 Terlalu Banyak Permintaan` dalam respons anda.

Jadual berikut menghuraikan had kadar untuk API yang berbeza:

| API           | Had Kadar tanpa Token     | Had Kadar dengan Token     |
| ------------- | ------------------------- | -------------------------- |
| Katalog Data  | 4 permintaan setiap minit | 10 permintaan setiap minit |
| OpenDOSM      | 4 permintaan setiap minit | 10 permintaan setiap minit |
| Amaran Banjir | 4 permintaan setiap minit | 10 permintaan setiap minit |
| Cuaca         | TBD                       | TBD                        |
| GTFS Statik   | TBD                       | TBD                        |
| GTFS Realiti  | TBD                       | TBD                        |

<span className="text-sm italic">* API yang belum dikeluarkan lagi ditandai dengan TBD.</span>

Sila ambil perhatian bahawa had kadar berbeza berdasarkan sama ada anda menggunakan token API atau tidak. Jika anda menjangkakan akan melebihi had kadar tanpa token API, kami mencadangkan anda [meminta token API](/authentication).
