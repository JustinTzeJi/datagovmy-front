## Had Pengehadan

Untuk memastikan penggunaan yang adil dan menjaga prestasi sistem, OpenAPI memperkenalkan had pengehadan pada bilangan permintaan yang boleh dibuat dalam jangka masa tertentu. Jika had tersebut dilanggar, permintaan seterusnya akan gagal, dan anda akan menerima ralat `429 Too Many Requests` dalam respons anda.

Jadual berikut menghuraikan had pengehadan untuk API yang berbeza dalam OpenAPI:

| API            | API Token Diperlukan | Had Pengehadan                    |
| -------------- | -------------------- | --------------------------------- |
| Data Catalogue | Tidak                | 4 permintaan sebanyak satu minit  |
| Data Catalogue | Ya                   | 10 permintaan sebanyak satu minit |
| GTFS-Static    | Tidak                | 4 permintaan sebanyak satu minit  |
| GTFS-Static    | Ya                   | 10 permintaan sebanyak satu minit |

Sila ambil perhatian bahawa had pengehadan berbeza bergantung kepada sama ada anda menggunakan token API atau tidak. Jika anda merancang untuk melampaui had pengehadan tanpa menggunakan token API, kami mengesyorkan agar anda [meminta token API](/overview/authentication).
